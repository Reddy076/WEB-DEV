# Semantic HTML

A semantic element clearly describes its meaning to both the browser and the developer.

**Non-semantic elements**: `<div>` and `<span>` - Tells nothing about its content.
**Semantic elements**: `<form>`, `<table>`, `<article>` - Clearly defines its content.

## Why use Semantic Elements?
1.  **Accessibility**: Screen readers can use semantic tags to help visually impaired users navigate a page ("Jump to navigation").
2.  **SEO**: Search engines give more weight to content inside semantic elements like headings, articles, and main content areas.
3.  **Readability**: Makes code easier to read and maintain for developers.

## Semantic Tags Layout

A standard HTML5 document layout typically looks like this:

```html
<header>...</header>
<nav>...</nav>
<main>
  <section>...</section>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>
```

### Element Descriptions

| Tag | Description |
|-----|-------------|
| `<header>` | Specifies a header for a document or section. |
| `<nav>` | Defines a set of navigation links. |
| `<section>` | Defines a section in a document. |
| `<article>` | Specifies independent, self-contained content (blog posts, news). |
| `<aside>` | Defines content aside from the page content (sidebars). |
| `<footer>` | Specifies a footer for a document or section. |
| `<main>` | Specifies the main content of a document. |
| `<figure>` | Specifies self-contained content, like illustrations, diagrams, photos. |
| `<figcaption>`| Defines a caption for a `<figure>` element. |
