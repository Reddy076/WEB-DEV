# Form Structure

An HTML form is used to collect user input. The user input is most often sent to a server for processing.

## The `<form>` Element
The `<form>` element defines the document section for user input.

```html
<form>
  <!-- Form elements -->
</form>
```

### Critical Attributes

| Attribute | Description |
|-----------|-------------|
| `action` | The URL where form data is sent on submission. |
| `method` | The HTTP method (`GET` or `POST`). |
| `enctype` | Encoding type (use `multipart/form-data` for file uploads). |
| `target` | Where to display the response associated with the result of submitting the form. |
| `autocomplete` | `on` or `off`. Specifies if browser should autocomplete. |

### GET vs POST
- **GET**: Appends form data to the URL `?name=value`. NEVER use for sensitive data. Good for search forms.
- **POST**: Sends data in the HTTP request body. Secure. No size limitations.

## Grouping Form Data
The `<fieldset>` element groups related data in a form.
The `<legend>` element defines a caption for the `<fieldset>`.
