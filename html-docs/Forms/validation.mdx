# Form Validation

HTML5 validation prevents data from being submitted if it does not meet certain criteria. This provides immediate feedback to the user.

## HTML5 Constraint Validation
Browsers have built-in validation mechanisms triggered by specific attributes.

| Attribute | Description |
|-----------|-------------|
| `required` | Specifies that an input field must be filled out. |
| `min` / `max` | Specifies a minimum and maximum value for numeric/date inputs. |
| `minlength` / `maxlength` | Specifies the minimum and maximum number of characters. |
| `pattern` | Specifies a regular expression that the input value must check against. |

## CSS Pseudo-Classes
You can style inputs based on their validity state:
- `:valid`: Selects elements with a valid value.
- `:invalid`: Selects elements with an invalid value.

```css
input:invalid {
  border: 2px solid red;
}
```
