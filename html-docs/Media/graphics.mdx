# Graphics

HTML5 allows you to draw graphics on your web page using `<canvas>` (raster graphics) and `<svg>` (vector graphics).

## SVG (Scalable Vector Graphics)
SVG defines vector-based graphics in XML format.
- SVG images are scalable without losing quality.
- Every element and attribute in SVG files can be animated.

```html
<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
</svg>
```

## Canvas
The `<canvas>` element is a container for graphics. You must use a script (usually JavaScript) to actually draw the graphics.

```html
<canvas id="myCanvas" width="200" height="100"></canvas>
```
Common methods: `getContext()`, `fillRect()`, `moveTo()`, `lineTo()`, `arc()`.

## Comparison
| Feature | Canvas | SVG |
|---------|--------|-----|
| Resolution | Resolution dependent (Raster) | Resolution independent (Vector) |
| Event Handlers | No support | Support |
| Complexity | Good for games/high particle count | Good for apps/maps (DOM accessible) |
| Rendering | Pixel-by-pixel | DOM based |
