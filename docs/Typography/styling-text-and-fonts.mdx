# Styling Text and Fonts

Text is the primary interface of the web. This section covers formatting text appearance.

## 1. Text Color
- `color`: Sets the foreground color of the text.

## 2. Text Alignment
- `text-align`: `left`, `right`, `center`, `justify`.
- *Note*: `justify` adds varying space between words to ensure lines end at the same point.

## 3. Text Decoration
Used for lines.
- `text-decoration: underline`: Standard link style.
- `text-decoration: none`: Removes underlines.
- `text-decoration: line-through`: Deleted text style.

## 4. Text Transformation
- `uppercase`: ALL CAPS.
- `lowercase`: no caps.
- `capitalize`: First Letter Of Each Word.

## 5. Spacing
- `letter-spacing`: Space between characters.
- `word-spacing`: Space between words.
- `line-height`: Vertical space. **Golden rule**: Use unitless numbers (e.g., `1.5` means 1.5x the font size).

## 6. Shadows
`text-shadow: offset-x offset-y blur color;`
```css
h1 {
  text-shadow: 2px 2px 5px rgba(0,0,0,0.5);
}
```

## 8. Important Interview Questions

<details>
  <summary>**1. Why use unitless `line-height`?**</summary>
  If you use `line-height: 20px` on a parent, and a child has `font-size: 30px`, the lines will overlap. If you use `line-height: 1.5`, the child creates a line height of `1.5 * 30px = 45px`, which scales correctly.
</details>

<details>
  <summary>**2. How to handle text overflow?**</summary>
  To get "..." (ellipsis):
  1. `white-space: nowrap;` (Force single line)
  2. `overflow: hidden;` (Hide extra)
  3. `text-overflow: ellipsis;` (Add dots)
</details>

<details>
  <summary>**3. How to remove the blue link underline?**</summary>
  `text-decoration: none;`.
</details>

<details>
  <summary>**4. Can you align text vertically using `text-align`?**</summary>
  No. `text-align` is strictly horizontal. Vertical alignment requires `line-height` (hacky), `vertical-align` (inline elements), or Flexbox/Grid (modern).
</details>

<details>
  <summary>**5. Difference between `word-break` and `overflow-wrap`?**</summary>
  `overflow-wrap: break-word` will break a long word only if it *has* to. `word-break: break-all` is more aggressive and will break words at random points to fit tightly.
</details>
