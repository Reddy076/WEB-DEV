# Transitions & Animations

Bringing the web to life.

## 1. Transitions
Smoothly change a property from State A to State B (e.g., Hover).
**Syntax**: `transition: property duration timing-function delay;`

```css
button {
  background: blue;
  transition: background 0.3s ease-in-out;
}
button:hover {
  background: red;
}
```

- **Timing Functions**: `linear`, `ease`, `ease-in`, `ease-out`, `cubic-bezier`.

## 2. Animations (@keyframes)
Complex multi-stage animations.

```css
@keyframes slideIn {
  0% { transform: translateX(-100%); }
  100% { transform: translateX(0); }
}

.box {
  animation: slideIn 1s forwards;
}
```

### Properties
- `animation-name`: Keyframe name.
- `animation-duration`: `2s`.
- `animation-iteration-count`: `infinite` or number.
- `animation-direction`: `alternate` (back and forth).

## 4. Important Interview Questions

<details>
  <summary>**1. Difference between Transition and Animation?**</summary>
  - **Transition**: Simple A to B change. Requires a trigger (hover, class change). 
  - **Animation**: Complex (A -> B -> C). Can start automatically (on load), loop, and pause.
</details>

<details>
  <summary>**2. Which properties are GPU accelerated (Performant)?**</summary>
  `transform` (translate, scale, rotate) and `opacity`. Animating `left/top` or `width/height` triggers layout recalculation (slow).
</details>

<details>
  <summary>**3. What is `will-change`?**</summary>
  A property to hint to the browser that an element is about to change, allowing it to optimize beforehand. Use sparingly.
</details>
