# Ways to apply CSS

There are three distinct methods to apply CSS to HTML. Each has a specific use case, though one is the industry standard.

## 1. External CSS (Recommended)

You create a separate `.css` file and link it in the HTML `<head>`.

**HTML:**
```html
<head>
  <link rel="stylesheet" href="styles.css">
</head>
```

**CSS (`styles.css`):**
```css
body {
  background-color: #f0f0f0;
}
```

### Why use it?
- **Global Scope**: One file changes the whole site.
- **Caching**: Loaded once, reused everywhere.
- **Clean Code**: Keeps HTML purely for structure.

## 2. Internal CSS

You write CSS inside a `<style>` tag within the HTML file, usually in the `<head>`.

**HTML:**
```html
<head>
  <style>
    h1 {
      color: green;
    }
  </style>
</head>
```

### Why use it?
- **Single Page Styles**: When a specific page needs unique overrides not relevant to the rest of the site.
- **Email Templates**: Email clients often require styles to be in the same file.

## 3. Inline CSS

You write CSS directly on the element using the `style` attribute.

**HTML:**
```html
<p style="color: blue; font-weight: bold;">I am blue.</p>
```

### Why use it?
- **Dynamic Styling**: JavaScript often applies animations or coordinates via inline styles.
- **Quick Testing**: Debugging a specific element.

### Why to avoid it?
- **Specificity Hell**: Hard to override (needs `!important`).
- **Maintenance**: Changes require finding every instance in HTML.
- **Payload**: bloating HTML size.

## 4. The `@import` Rule

A way to load CSS *from inside* CSS.

```css
@import url("navigation.css");
@import url("footer.css");
```

*Performance Note: Using multiple `@import` rules can create a "waterfall" effect where files load one by one, slowing down the site. `<link>` is preferred.*
